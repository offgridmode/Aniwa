# Aniwa v0.1.1

> Configuration System & Workflow Improvement Release

Release Date: May 16, 20206

---

# Overview

v0.1.1 represents the first major improvement release for Aniwa following the foundational v0.1.0 release.

This release primarily focuses on:

- configuration systems
- workflow improvements
- output management
- profiling usability
- architectural refinement
- developer experience

The goal of this release is to make Aniwa more:

```text
reusable, automation-friendly, and scalable
```

for modern data workflows.

---

# Release Goals

The primary goals of v0.1.1 were:

- introduce configuration workflows
- improve profiling usability
- strengthen architecture
- improve developer experience
- prepare the ecosystem for future scalability

---

# Major Features

---

# Configuration File Support

v0.1.1 introduces the first official configuration system for Aniwa.

---

# Why Configuration Matters

Before this release, users needed to repeatedly provide CLI arguments manually.

Configuration files now enable:

- reusable workflows
- reproducibility
- CI/CD integration
- automation-friendly execution
- team-wide consistency

---

# Supported Configuration Formats

Aniwa now supports:

| Format |
|---|
| YAML |
| TOML |
| JSON |

---

# Automatic Config Discovery

Aniwa automatically searches for:

```text
aniwa.yaml
aniwa.yml
aniwa.toml
aniwa.json
```

within the current working directory.

---

# Example Configuration

Example:

```yaml
mode: deep

report:
  format: html
  output_dir: reports/

sections:
  include:
    - summary
    - schema
    - statistics
    - insights
```

---

# Custom Config File Support

Users can now load custom configuration files using:

```bash
aniwa customers.csv --config config.yaml
```

---

# Configuration Priority Resolution

Aniwa now follows:

```text
CLI > config file > defaults
```

---

# Why Priority Resolution Matters

This enables:

- predictable execution
- flexible overrides
- automation safety

---

# Configuration Validation

v0.1.1 introduces improved validation for:

- invalid profiling modes

---

# Helpful Error Handling

Configuration systems now provide:

```text
clear actionable error messages
```

# Improved Output Management

v0.1.1 introduces improved report output handling.

# Output Directory Support

Users can now generate reports into directories using:


# Automatic File Naming

Aniwa now automatically generates default report names such as:

| Format | Default Output |
|---|---|
| HTML | aniwa_report.html |
| JSON | aniwa_report.json |
| Markdown | aniwa_report.md |

Aniwa now automatically creates missing output directories when needed.
